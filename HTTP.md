# HTTP
- HTTP란 웹 상에서 데이터를 주고 받기 위한 프로토콜이다.
- HTTP는 클라이언트와 서버간의 ``요청과 응답``을 통해 작동한다.
- HTTP는 다양한 데이터을 전송할 수 있도록 설계된 프로토콜이다.
- HTTP는 ``단방향성``이다.


# <h2> HTTP의 장점
- 구조가 단순하여 구현이 쉽고 다양한 데이터 형식을 주고 받을 수 있다.

- 웹 브라우저뿐만 아니라 다양한 기기에서도 호환성이 높다.

- 빠른속도






# <h2> HTTP의 단점
- HTTP의 단점은 보안이 약하다는 것이다.
- HTTP의 단점을 보안하기 위해 나온 것이 ``HTTPS``이다.

# <h2> HTTP 요청 메서드

- GET: 정보를 요청하기 위해 사용한다.
- HEAD GET: body의 정보 말고 HEAD정보를 요청하기 위해 사용한다.
- POST: 서버에 정보를 ``전송(추가)``할 때 사용한다.
- PUT: 정보를 ``갱신, 수정``할 때 사용한다.
- DELETE: 정보를 ``삭제``하기 위해서 사용한다.
- OPTIONS: 서버에서 지원하는 HTTP 메서드의 목록을 요청할 때 사용.
- PATCH: 리소스 일부분을 수정.

# <h2> PUT과 PATCH의 차이점

PUT은 ``전체``를 수정하지만<br>PATCH는 ``일부분``만 수정한다는 점이다.


# <h2> HTTP와 HTTPS의 차이점

HTTP는 데이터를 그대로 보내기 때문에 중간에 누가 나쁜짓을 할 수 있는데<br>HTTPS는 암호화를 사용해서 데이터를 암호화 한 뒤 보내기 때문에 누가 나쁜 짓을 할 수 없다.

**HTTP와 HTTPS의 핵심 차이는 보안이다.**


# <h2> HTTP 무상태성

HTTP의 무상태성은 클라이언트가 서버에 요청을 보내는 동안
서버가 클라이언트의 상태를 ``저장하지 않는다``라는 뜻이다.

# <h2> HTTP 상태코드

HTTP의 상태코드는 클라이언트의 요청에 대한 서버에 응답 상태를 나타내는<br> ``3자리 숫자 코드``이다.
