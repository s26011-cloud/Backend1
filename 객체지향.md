# 객체지향
프로그램을 데이터와 절차의 단순 나열이 아닌 <br> 독립적인 ``객체(Objeck)``들의 모임과 상호작용으로 파악하는 프로그래밍 패러다임.

# <h2> 캡슐화
데이터와 그 데이터를 처리하는 메서드를 하나로 묶고<br>외부의 직접적인 접근을 제한하는 기법.

# <h2> 상속
기존 클래스의 기능을 물려받아서 새로운 클래스를 만드는 것.

# <h2> 다형성
하나의 객체(또는 메서드)가 여러 형태로 동작하는 것.

# <h2> 추상화
클래스들의 공통적인 요소를 뽑아서 상위 클래스를 만들어내는 것이다.

#  <h2> 객체지향의 장단점

 # <h3>장점
- 코드 유지 보수 및 확장이 쉽다.

- 코드 재사용이 용이하다.

- 사람이 많은 대형 프로젝트에 적합하다.

# <h3>단점
- 설계 시 많은 노력이 요구된다.

- 절차지향에 비해 실행속도가 느리며<br>필요한 메모리 양이 많습니다.

# <h2> 관련 코드

```class Dog {
    void bark() {
        System.out.println("멍멍");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog(); // 객체 생성
        d.bark();          // 메서드 호출
    }
}
