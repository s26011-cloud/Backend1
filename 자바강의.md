# - 클래스와 객체

# <h2> 클래스
자바는 객체를 만들기 위해 반드시 클래스를 먼저 만들어야 한다. 클래스는 객체를 만들기 위한 일종의 ``틀``이다
``` 
public class Car{

}
```
# <h2> String클래스
``문자열을 표현``하는 자바에서 가장 많이 사용하는 클래스

```
String str1 = "hello";
String str2 = "hello";
```
# <h2> 필드(field)선언
자바에서 ``속성``을 ``필드(field)``라는 용어로 사용한다.
```
public class Car{
        String name;    
        int number;
    }
```
# <h2> 메소드란?
필드가 물체의 상태라면, 물체의 ``행동``에 해당하는게 메소드.

# <h2>변수의 scope와 static
프로그램상에서 사용되는 변수들은 사용 가능한 범위를 가진다. 그 ``범위``를 변수의 ``scope``라고 한다.

static은 자바에서 "정적"을 의미하며, 클래스에 ``고정된 멤버를 정의``할 때 사용된다.

# <h2> 열거형(enum) 
자바는 열거타입을 이용하여 변수를 선언할 때 변수타입으로 사용할 수 있다.


# - 클래스 다듬기


# <h2>생성자
모든 클래스는 인스턴스화 될때 생성자를 사용한다.

# <h3>생성자의 특징

- 생성자는 리턴타입이 없다.

- 생성자를 프로그래머가 만들지 않으면 매개변수가 없는 생성자가 컴파일할 때 자동으로 만들어진다.

- 매개변수가 없는 생성자를 기본생성자랄고 한다.

- 생성자를 하나라도 프로그래머가 만들었다면 기본생성자는 자동으로 만들어지지 않는다.

# <h2> this
this는 현재 객체, 자기 자신을 나타낸다.

```
public class Car{
        String name;
        int number;

        public Car(String n){
            name = n;
        }
    }
```

# <h2>메소드 오버로딩
매개변수의 유형과 개수가 다르게 하여 같은 이름의 메소드를 여러 개 가질 수 있게하는 기술.

```
class MyClass2{
        public int plus(int x, int y){
            return x+y;
        }

        public int plus(int x, int y, int z){
            return x + y + z;
        }

        public String plus(String x, String y){
            return x + y;
        }
    }
```

# <h2>패키지
패키지란 서로 관련이 있는 클래스 또는 인터페이스들을 ``묶어 놓은 묶음``이다.

# - 상속

# <h2>상속
상속이란 부모가 가진것을 자식에게 물려주는것을 의미한다.

# <h2>접근제한자
접근 제한자란 클래스 내에서 멤버의 접근을 제한하는 역할을 한다.

# <h2>추상 클래스
추상 클래스란 구체적이지 않은 클래스를 의미한다.

# <h2> super와 부모생성자

# <h3>부모생성자
부모의 생성자부터 실행된다.

# <h3>super
자신을 가리키는 키워드가 this 라면, 부모을 가리키는 키워드는 ``super``이다.

```
public Bus(){
        super();
        System.out.println("Bus의 기본생성자입니다.");
    }
```

# <h2>오버라이딩
오버라이딩이란 부모가 가지고 있는 메소드와 ``똑같은 모양의 메소드``를 자식이 가지고 있는 것이다.

# <h2>클래스 형변환
자식객체가 가지고 있는 메소드나 속성을 사용하고 싶다면 형변환 해야 한다.


# <h2>인터페이스
인터페이스: 서로 관계가 없는 물체들이 상호 작용을 하기 위해서 사용하는 장치나 시스템.

# - 예외처리

# <h2>Exception
프로그램실행중 예기치 못한 사건을 예외라고 한다. 예외 상황을 미리 예측하고 처리할 수 있는데, 이렇게 하는 것을 ``예외 처리``라고 한다.

# <h2>Throws
throws는 예외가 발생했을때 예외를 호출한 쪽에서 처리하도록 던져준다.
