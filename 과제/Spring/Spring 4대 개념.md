# Spring 4대 개념

## DI란?
- 의존 관계 주입이라고도 하며, 어떤 객체가 사용하는 의존 객체를 직접 만들어 사용하는게 아니라, ``주입`` 받아 사용하는 방법이다.

### DI 주요 기능 및 특징
- 객체 간의 결함도 감소

- 자동 의존성 주입

- 객체 생명주기 관리

- 테스트 용이성

## IoC란?
- IoC(제어 반전)이란 객체의 ``생성``, ``생명주기의 관리``까지 모든 객체에 대한 제어권이 바뀌었다는 것을 의미합니다.

### IoC 컨테이너
- 컨테이너는 보통 객체의 ``생명주기``를 관리,생성된 인스턴스에게 ``추가적인 기능``을 제공하도록 하는 것이다.

- 스프링 프레임워크도 객체를 ``생성``하고 ``관리``하고 `책임`지고 `의존성`을 관리해주는데 그게 IoC 컨테이너이다.

### IoC 핵심 기능 및 특징
- 객체 생성 및 관리

- 의존성 주입

- 제어 흐름의 역전

- 컴포넌트 스캔

## PSA란?
- 환경의 변화와 관계없이 일관된 방식의 기술로의 접근 환경을 제공하는 ``추상화 구조``이다.

### 핵심 기능 및 특징
- 일관된 기술 접근

- 높은 유지보수성 및 포터빌리티

- POJO 기반의 추상화

#### 포터빌리티(Portability)란
- 기기나 데이터, 소프트웨어가 다양한 환경에서 편리하게 이동하거나 사용될 수 있는 능력

## AOP란?
- 공통적으로 등장하는 기능(관심사)을 ``분리하여 모듈화``하는 기술이다.

### 핵심 기능 및 특징
- 프록시 기반 AOP

- 메서드 조인 포인트만 지원

- 런타임

- Spring 컨테이너와의 통합
